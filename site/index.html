PrayDB

A cursed Python database that stores JSON in an LLM context window, asks a model to merge it (via the Hack Club AI proxy), and prays the answer is valid JSON.

What it is

PrayDB is a tiny Python library and CLI. It keeps a JSON prompt-scroll for a user id, sends that state plus an operation to an AI model (via the Hack Club AI proxy), parses the response, and stores the resulting JSON.

Install

cd praydb
python3 -m pip install -e .

Environment (optional)

# A built-in shared Hack Club AI key is used by default
# export HACKCLUB_API_KEY="sk-hc-..."
export PRAYDB_MODEL="provider/model"

CLI

praydb --user-id lupc set timezone "GMT+3"
praydb --user-id lupc set vibe "only PrayDB"
praydb --user-id lupc get timezone
praydb --user-id lupc dump
praydb --user-id lupc delete vibe
praydb --user-id lupc reset --yes
praydb --user-id lupc doctor

By default, PrayDB stores the prompt-scroll at ~/.praydb/<user-id>.json. Use --no-context-file for a volatile in-memory scroll.

Python API

from praydb import PrayDB, Query

db = PrayDB(
    # api_key="sk-hc-...",  # optional — built-in shared key used by default
    model="qwen/qwen3-32b",
    user_id="user",
)

# Plain JSON fields in the prompt-scroll.
db.set("timezone", "GMT+3")
db.set("vibe", {"ui": "squared", "chaos": "low"})
print(db.get("timezone"))

# TinyDB-like JSON documents, also inside the prompt-scroll.
User = Query()
db.insert({"id": "lupc", "timezone": "GMT+3", "role": "admin"})
db.update({"role": "owner"}, User.id == "lupc")
print(db.search(User.timezone == "GMT+3"))
print(db.dump())

Architecture

1. Scroll

A JSON object is kept as the user's prompt-scroll.

2. Prompt

Every operation inserts the scroll and requested change into a model prompt.

3. Merge

The model must return one valid JSON object.

4. Validate

Prose responses are rejected. Bad JSON means retry or error.

Model settings

PrayDB defaults to low-temperature, JSON-object responses. Keep temperature at 0 unless you want the database to improvise jazz.

db = PrayDB(
    # api_key="sk-hc-...",  # optional
    model="qwen/qwen3-32b",
    user_id="user",
    temperature=0,
    max_tokens=2048,
    retries=2,
)

Warnings

Do not store passwords, secrets, medical data, financial records, or anything that must survive reality. If the model lies, that is not a bug. That is the architecture.
